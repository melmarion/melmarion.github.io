I Calibrated a Persuasion Model on 126,288 Human Interactions. Here’s What the Data Actually Says.

Most persuasion advice is folk wisdom dressed up as science. I built a system that checks.

There’s a genre of advice that sounds like this: “Use social proof. Appeal to authority. Create urgency.” It comes from textbooks, TED talks, and LinkedIn posts with 40,000 likes. It sounds right. It feels scientific. And it treats persuasion like a universal lever — push it, and people comply.

I built a system to test that assumption empirically. The short answer is that it’s wrong. The same technique, applied to two different people, can produce a 21.8 percentage point difference in compliance. Persuasion isn’t a lever. It’s a lock, and the key changes depending on who’s holding the door.

The system is called Persuasion-Max. It’s open source. Here’s what it does, how it works, and what the data revealed.

The architecture

Persuasion-Max is a 5-layer pipeline. Each layer transforms the input into a richer representation, and each layer was built because the previous one wasn’t sufficient.

01 Linguistic Surface 12 features
02 Cognitive Appraisal 7 dimensions
03 Technique Detection 40-technique Zeng taxonomy
04 Recipient Modeling 16-dimension personality profile
05 Domain Circuit Prediction ecommerce, campaign, crisis PR

The first layer extracts surface-level linguistic features: reading difficulty, emotional valence, pronoun ratios, sentence complexity. The kind of thing you’d get from a sentiment analysis tool, but tuned for persuasion-relevant signals.

The second layer maps text onto cognitive appraisal dimensions — a framework from Smith & Ellsworth (1985) that describes how humans evaluate incoming information along axes like novelty, certainty, goal relevance, coping potential, and agency. This is the layer that captures how the message makes you feel before you’ve consciously processed it.

The third layer identifies which of 40 named persuasion techniques are present, using the Zeng taxonomy from SemEval. Anchoring. Social proof. Loss framing. Emotional appeal. Bandwagon. Each one is detected and scored.

The fourth layer is where it gets interesting. Instead of treating the recipient as a generic “human,” this layer builds a 16-dimension profile: Big Five personality traits, Moral Foundations (Haidt), ideology markers, and cognitive style. The same message hits differently depending on who receives it. This layer models that.

The fifth layer combines everything into domain-specific behavioral predictions. Because persuasion in ecommerce is mechanistically different from persuasion in political campaigns, which is different from crisis PR. Cross-domain transfer doesn’t work — and I have the numbers to prove it.

The calibration dataset combines four sources: DailyPersuasion (78K dialogues across 35 domains), HumanChoicePrediction (48K real binary decisions where humans chose between competing messages), PersuGPT, and PERSUADE 2.0. Every parameter in the system has a provenance tag: 1% FITTED on real data, 61% CONSTRAINED from published literature, 37% UNCALIBRATED with documented assumptions.

I track this because hand-assigned numbers are the silent killer of behavioral models. If you don’t know which weights came from data and which came from intuition, you don’t know what your model actually learned.

Finding 1: Linguistic features matter more than you’d think

Linguistic surface features contribute +10 percentage points AUC over cognitive appraisal alone. The way you say it matters as much as what you say. Sentence structure, reading level, pronoun choice — these aren’t noise. They’re signal.

This was counterintuitive. The appraisal layer captures meaning. The linguistic layer captures form. I expected meaning to dominate. Instead, adding surface features on top of appraisal produced a consistent, significant improvement in prediction accuracy. The implication is that persuasion operates on multiple cognitive channels simultaneously — you’re processing what someone is saying and how they’re saying it, and both contribute to whether you comply.

Finding 2: Technique detection adds zero incremental signal

Binary technique detection (is anchoring present: yes/no) adds zero incremental AUC. Knowing which techniques are used, without knowing who they’re used on, has no predictive value.

This one surprised me. The entire persuasion advice industry is built on technique identification: learn these 12 techniques and you’ll be more persuasive. But when I added technique detection as a binary feature to the model, it contributed nothing. Zero. The model already had all the signal it needed from linguistic and appraisal features.

The techniques only become predictive when you cross them with recipient personality — which leads to the main finding.

Finding 3: The 21.8pp persona compliance spread

The same persuasion technique produces up to a 21.8 percentage point difference in compliance probability depending on recipient personality. Persuasion is not universal. It is recipient-dependent.

I built a 400-cell interaction matrix crossing 40 techniques against 10 persona archetypes. The spread is enormous. Here’s a sample:

Technique High Openness High Conscientiousness Spread
Storytelling +14.2pp +1.8pp 12.4pp
Evidence-based framing +3.1pp +18.6pp 15.5pp
Moral reframing +11.3pp -2.1pp 13.4pp
Loss framing +0.4pp +16.9pp 16.5pp

Storytelling works on people high in openness. Evidence works on people high in conscientiousness. Moral reframing — matching the message to the recipient’s moral foundations (Feinberg & Willer, 2015) — produces +8.4pp when the foundation is correctly matched and can actually backfire when mismatched. The technique isn’t good or bad. It’s good for someone.

Finding 4: Domain specificity is real

Cross-domain weight transfer degrades AUC by 10–13 percentage points. A persuasion model trained on ecommerce data performs significantly worse on political campaigns, and vice versa.

This confirms what practitioners intuit but the field largely ignores: persuasion mechanics are context-dependent. The features that predict purchase behavior are not the same features that predict vote change. The system maintains separate domain-specific weight sets for this reason.

One domain-specific finding worth highlighting: in crisis PR, defensive techniques like whataboutism trigger a 46.5% retaliation probability. The technique that feels protective is actually accelerating the crisis.

Finding 5: Authenticity outperforms strategy

This one wasn’t in the calibration data. It emerged from a companion project — Read the Room, an iOS app where you practice influence techniques on 14 AI characters with full psychological profiles.

The last character is a therapist. I gave her no defense mechanisms. Instead, her profile reads: “None — she detects yours.” Every named technique in the taxonomy fails on her. The only responses that score points are authentic ones.

I didn’t design that as a philosophical statement. It fell out of the scoring model. Forty techniques, tested against fourteen personality profiles, and the architecture arrived at a conclusion: when the recipient has high psychological sophistication, technique detection flips from asset to liability. The recipient recognizes the technique, attributes manipulative intent, and compliance drops below baseline.

The math says authenticity isn’t a moral choice. It’s a strategy that dominates when the other person is paying attention.

What this is for

Persuasion-Max isn’t a tool for manipulating people. It’s a diagnostic tool for understanding why messages work or don’t work, and for building products that communicate more effectively with diverse audiences.

If you’re writing onboarding copy and different user segments respond differently, this system can tell you why. If you’re building a recommendation engine and need to understand how framing affects choice, this system models that. If you’re doing trust & safety work and need to detect influence techniques in the wild, the technique detection layer is a starting point.

The full system includes 17 FastAPI endpoints, 12 MCP tools for Claude integration, an interactive SVG brain visualization, and 329 automated tests. Every parameter has a provenance tag. The weight registry is auditable.

The source code, calibration data, and full documentation are on GitHub.
